# CS入门科普 - 部署指南

本项目是一个纯前端静态网站，包含5个交互式教学页面。以下是多种部署方式供您选择。

## 📁 项目文件结构

```
科普/
├── index.html          # 首页（导航页）
├── 计算机硬件.html      # 硬件架构课程
├── 软件.html           # 软件工具课程  
├── 网络.html           # 计算机网络课程
├── LLM.html            # 大语言模型课程
├── os.html             # 操作系统课程
└── README.md           # 本文件
```

## 🚀 部署方式

### 方式 1: 本地打开（最简单）

**适用场景**: 个人学习、快速预览

**步骤**:
1. 确保所有 `.html` 文件在同一文件夹
2. 双击 `index.html` 文件
3. 在浏览器中打开即可使用

**优点**: 零配置，无需服务器  
**缺点**: 无法在线分享给他人

---

### 方式 2: GitHub Pages（推荐 - 免费托管）

**适用场景**: 在线分享、作品展示

**步骤**:

#### A. 使用 GitHub Desktop（图形化界面）

1. **安装 GitHub Desktop**
   - 访问 https://desktop.github.com/
   - 下载并安装

2. **创建仓库并上传**
   - 打开 GitHub Desktop
   - 点击 `File` → `New Repository`
   - Repository Name: `cs-tutorial`（或任意名称）
   - Local Path: 选择包含HTML文件的文件夹的**上一级**目录
   - 点击 `Create Repository`

3. **发布到 GitHub**
   - 点击右上角 `Publish repository`
   - 取消勾选 `Keep this code private`（设为公开）
   - 点击 `Publish Repository`

4. **启用 GitHub Pages**
   - 在 GitHub Desktop 中点击 `Repository` → `View on GitHub`
   - 在网页中点击 `Settings`
   - 左侧菜单找到 `Pages`
   - Source 选择 `main` 分支，文件夹选 `/(root)`
   - 点击 `Save`
   - 等待 1-2 分钟后，页面会显示网站地址，如: 
     ```
     https://你的用户名.github.io/cs-tutorial/
     ```

5. **访问网站**
   - 直接访问上述地址即可查看首页
   - 页面间通过导航栏切换

#### B. 使用命令行（Git）

```bash
# 1. 进入项目文件夹
cd /path/to/科普

# 2. 初始化 Git 仓库
git init
git add .
git commit -m "Initial commit"

# 3. 在 GitHub 网站创建新仓库（不要初始化 README）

# 4. 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/cs-tutorial.git

# 5. 推送代码
git branch -M main
git push -u origin main

# 6. 在 GitHub 仓库设置中启用 Pages（参考上述步骤4）
```

**优点**: 
- 免费托管
- 自动 HTTPS
- 可以分享给任何人

**缺点**: 
- 需要 GitHub 账号
- 首次配置稍复杂

### 方式 3: Vercel/Netlify（一键部署）

**适用场景**: 快速部署、自动更新

#### Vercel 部署步骤:

1. **准备工作**
   - 注册 Vercel 账号 (https://vercel.com)
   - 确保项目已上传到 GitHub

2. **导入项目**
   - 登录 Vercel
   - 点击 `New Project`
   - 选择 `Import Git Repository`
   - 授权并选择你的仓库

3. **配置部署**
   - Framework Preset: `Other`
   - Root Directory: `./` (根目录)
   - 点击 `Deploy`

4. **获取链接**
   - 部署完成后会得到一个 `.vercel.app` 域名
   - 每次 Git 推送都会自动重新部署

#### Netlify 部署步骤:

1. **拖拽上传**（最简单）
   - 访问 https://app.netlify.com/drop
   - 将整个文件夹拖入浏览器窗口
   - 自动部署并生成链接

2. **GitHub 连接**（自动部署）
   - 登录 Netlify
   - `New site from Git`
   - 选择 GitHub 仓库
   - 自动检测为静态网站
   - 点击 `Deploy`

**优点**: 
- 全球 CDN 加速
- 自定义域名支持
- 自动 HTTPS
- CI/CD 自动部署

**缺点**: 
- 免费版有流量限制（但个人使用完全够用）

### 方式 4: 本地 HTTP 服务器（开发测试）

**适用场景**: 本地开发、多设备访问

#### Python 3

```bash
# 进入项目文件夹
cd /path/to/科普

# 启动服务器
python -m http.server 8000

# 浏览器访问: http://localhost:8000
```

#### Node.js (http-server)

```bash
# 全局安装
npm install -g http-server

# 进入项目文件夹并启动
cd /path/to/科普
http-server -p 8000

# 浏览器访问: http://localhost:8000
```

#### PHP

```bash
cd /path/to/科普
php -S localhost:8000
```

#### VS Code Live Server 插件

1. 安装 "Live Server" 扩展
2. 右键 `index.html`
3. 选择 "Open with Live Server"
4. 自动打开浏览器

**优点**: 
- 支持局域网访问
- 热重载（Live Server）
- 适合开发调试

**缺点**: 
- 需要保持终端运行
- 仅限局域网访问

### 方式 5: 云服务器部署（专业方案）

**适用场景**: 生产环境、企业使用

#### Nginx 配置示例:

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    
    root /var/www/cs-tutorial;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    # 启用 Gzip 压缩
    gzip on;
    gzip_types text/html text/css application/javascript;
}
```

**部署步骤**:
```bash
# 1. 上传文件到服务器
scp -r ./科普/* user@server:/var/www/cs-tutorial/

# 2. 配置 Nginx（参考上面配置）
sudo nano /etc/nginx/sites-available/cs-tutorial

# 3. 启用站点
sudo ln -s /etc/nginx/sites-available/cs-tutorial /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 4. 配置 SSL（可选）
sudo certbot --nginx -d yourdomain.com
```

## 🛠️ 故障排除

### 问题 1: 页面间导航不工作

**原因**: 文件名或路径不匹配  
**解决**:
- 确保所有 HTML 文件在同一文件夹
- 检查文件名是否有中文字符（建议重命名为英文）
- 如果部署到服务器，确保 URL 编码正确

### 问题 2: 样式显示异常

**原因**: CDN 资源加载失败  
**解决**:
- 检查网络连接
- 尝试刷新页面（Ctrl+F5 强制刷新）
- 如果在内网环境，需要配置代理

### 问题 3: GitHub Pages 404 错误

**原因**: Pages 设置未正确配置  
**解决**:
- 确认 Settings → Pages 中 Source 设置为 `main` 分支
- 等待 1-5 分钟让 GitHub 构建完成
- 访问 `https://用户名.github.io/仓库名/index.html`

## 📊 性能优化建议

### 1. 启用浏览器缓存

在 Nginx 配置中添加:
```nginx
location ~* \.(html|css|js)$ {
    expires 7d;
    add_header Cache-Control "public, immutable";
}
```

### 2. 压缩资源

如果自建服务器，启用 Gzip:
```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/html text/css application/javascript;
```

### 3. CDN 加速

对于高访问量场景，考虑使用:
- Cloudflare (免费 CDN)
- 腾讯云 CDN
- 阿里云 CDN
